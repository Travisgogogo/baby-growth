## 3. 发现的问题

### 🔴 严重问题 (需立即修复)

#### 3.1 数据库连接未正确关闭
**位置**: database_service.dart
**问题**: 虽然提供了 close() 方法，但应用生命周期中没有调用
**风险**: 应用退出时数据库连接可能泄漏
**建议**: 在应用 dispose 时关闭数据库

#### 3.2 图片文件路径存储可能存在安全风险
**位置**: 多个模型文件
**问题**: 直接使用文件路径字符串存储，未验证路径合法性
**风险**: 如果路径被篡改，可能访问非预期文件
**建议**: 使用应用沙箱目录，验证路径前缀

### 🟡 中等问题 (建议修复)

#### 3.3 部分异步操作未处理异常
**位置**: 
- home_screen.dart: _loadData() 中部分操作
- cloud_backup_screen.dart: WebDAV 操作
**问题**: 部分 async 调用没有 try-catch
**风险**: 可能导致未捕获的异常，影响用户体验

#### 3.4 长函数/大文件
**位置**:
- home_screen.dart: 超过1000行
- health_screen.dart: 超过700行
- profile_screen.dart: 超过500行
**问题**: 单个文件职责过多，维护困难
**建议**: 按功能拆分成多个小文件

#### 3.5 硬编码字符串和魔法数字
**位置**: 多处
**示例**:
```dart
// 硬编码的年龄阈值
if (ageInMonths < 1) return 45;
if (ageInMonths < 3) return 60;
// ...
```
**建议**: 提取为常量或配置

#### 3.6 重复代码
**位置**:
- 日期格式化逻辑在多个文件中重复
- 图片选择逻辑重复
**建议**: 提取到工具类

### 🟢 轻微问题 (可选优化)

#### 3.7 缺少单元测试
**现状**: 无测试文件
**建议**: 为核心业务逻辑添加单元测试

#### 3.8 部分注释缺失
**位置**: 复杂算法和工具函数
**建议**: 添加文档注释

#### 3.9 未使用的导入
**位置**: 部分文件存在未使用的 import
**建议**: 定期清理

## 4. 潜在风险和隐患

### 4.1 数据丢失风险
- **问题**: 云端备份依赖用户手动操作，无自动备份
- **影响**: 用户可能忘记备份，换手机时数据丢失
- **建议**: 添加自动备份选项，或定期提醒

### 4.2 版本兼容性
- **问题**: 数据库 migration 逻辑较简单，复杂升级可能出错
- **影响**: 版本升级时可能丢失数据
- **建议**: 添加更健壮的 migration 测试

### 4.3 WebDAV 依赖
- **问题**: 云端备份仅支持坚果云 WebDAV
- **影响**: 用户选择受限，坚果云服务变化会影响功能
- **建议**: 考虑支持更多云存储选项

### 4.4 图片存储
- **问题**: 图片直接存原图，无压缩策略
- **影响**: 长期使用可能占用大量存储空间
- **建议**: 添加图片压缩和清理机制

## 5. 性能优化建议

### 5.1 列表优化
- 大数据量列表使用 ListView.builder 已正确实现 ✅
- 建议: 添加分页加载，避免一次性加载过多记录

### 5.2 图片加载
- 当前使用 Image.file 直接加载
- 建议: 使用 cached_network_image 或类似库缓存缩略图

### 5.3 数据库查询
- 当前查询无索引优化
- 建议: 为常用查询字段添加索引（如 babyId, date 等）

### 5.4 状态管理
- 当前使用 setState，页面复杂时可能重建过多
- 建议: 考虑使用 Provider 或 Riverpod 优化状态管理
